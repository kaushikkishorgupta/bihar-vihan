Role Policy

  • Only users listed in `admin_users/{uid}` can access live controls.
  • All writes go to Firestore collections for posts, explore cards, ads, and media.
  • Image uploads use Firebase Storage folder `media/`.

Create Additional Admin

Only logged-in admin can create additional admins.

How It Works

  • Creates a new Firebase Authentication user.
  • Adds role document in `admin_users/{uid}` with `role: admin`.
  • New admin can login from this panel and edit content.

Roles & Access Control

Core Admin Roles

Super AdminFull system control. Manage admins and all users. Access all settings and analytics. Delete or modify any content.
AdminManage users (except Super Admin). Manage content and media. Approve or reject posts. Manage website settings.
ModeratorApprove or reject comments and posts. Remove spam or inappropriate content. Flag users if needed.
EditorEdit and publish content. Manage categories and tags. Update existing posts.
AuthorCreate and publish own posts. Edit own content only.
ContributorCreate content and submit for approval. Cannot publish directly.

Support Roles

SEO ManagerEdit meta tags, titles, and descriptions. Manage sitemap and keywords. Monitor SEO performance.
Content ManagerUpload and manage media files. Organize blogs and galleries. Schedule content publishing.
Support AgentRespond to user queries. Manage contact form messages. Handle complaints and feedback.
AnalystView traffic and performance reports. Access analytics dashboard. Export reports (read-only).
